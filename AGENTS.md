# AGENTS.md

## 1. Назначение

Документ определяет правила для AI-агентов и разработчиков.

Цель:
- читаемые тесты
- простая архитектура
- минимальная стоимость поддержки

Если изменение не улучшает ничего из этого — его не нужно делать.

---

## 2. Архитектура

Строгое разделение слоёв:

tests → pages/api → core

### Пакеты

- tests  
  Только тесты (сценарии)

- pages / api  
  Page Objects и API-клиенты (действия)

- core  
  драйвер, конфигурация, базовая инфраструктура

### Правила

- Тесты НЕ обращаются к core
- Тесты НЕ содержат локаторов
- Page Objects НЕ содержат ассертов (кроме isOpened/isLoaded)
- В core нет бизнес-логики

---

## 3. Тесты

### Требования

- Тест = сценарий пользователя
- Название = бизнес-смысл
- Ассерты только в тестах

### Запрещено

- Thread.sleep / Selenide.sleep
- new WebDriver
- if / switch
- циклы
- вычисления
- ассерты вне тестов

---

## 4. Page Objects / API

### Разрешено

- действия (click, type, select)
- навигация
- isOpened / isLoaded

### Запрещено

- ассерты
- бизнес-логика
- условия
- вычисления
- лишние абстракции

---

## 5. Конфигурация

Разрешено:
- *.properties

Запрещено:
- хардкод URL, кредов, селекторов
- конфиг в коде

---

## 6. Антипаттерны

Исправляются сразу:

- sleep
- ручной WebDriver
- XPath без необходимости
- God-классы
- универсальные методы с множеством параметров
- логика в Page Object

---

## 7. Правила для AI

Обязан:
- соблюдать архитектуру
- писать максимально просто
- уменьшать дублирование

Запрещено:
- добавлять новые абстракции
- делать "умные" обёртки
- добавлять retry/магии
- вводить новые слои конфигурации

---

## 8. Проверка результата

Перед завершением:

- все тесты проходят
- нет флаки
- проект собирается
- checkstyle/линтер проходят
- нет неиспользуемого кода
- код понятный(быстро и легко читается)

---

## 9. Definition of Done

Изменение принимается, если:

- тест читается как сценарий
- нет хардкода
- нет sleep
- нет лишней сложности
- архитектура не нарушена
- все проверки из пункта 8 пройдены

---

## 10. Финальное правило

Если есть сомнение:
выбирай самое простое решение без абстракций.
